Em q10_resposta.txt: identifique cada problema e explique por que é um problema.
Dica: há um problema em: (1) definição da função, (2) como o loop é escrito, (3) como a lista é construída.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
SUA RESPOSTA:

(1) definição da função:

Problema identificado em " def processar_alunos(alunos=[]): " foi o parâmetro "alunos" da função recebendo uma lista padrão como argumento em caso de nenhum argumento/valor ser passado ("alunos=[]").
É um problema porque esse tipo de abordagem gera erro de vazamento de dados, já que a lista é um argumento padrão mutável, onde a partir da 2ª chamada da função o argumento pode ser o mesmo da 1ª chamada.
Dependendo de como o código estiver escrito, uma alteração na lista na 2ª chamada da função pode gerar uma alteração indesejada na lista que foi usada na 1ª chamada da função, gerando inconsistências. Outra
forma de contornar isso seria atribuir por padrão à alunos a palavra reservada None, dessa forma: " def processar_alunos(alunos=None): "

(2) como o loop é escrito: 

Problema identificado em "for i in range(len(alunos)):" foi o uso desnecessário do índice i, o que por consequência força o uso de len(alunos) como range do laço de repetição. É um problema
porque criar uma variável e um processamento que consome recurso computacional da máquina, podendo ser otimizado da seguinte forma já que o interesse único é pelos valores da lista de alunos:
"for aluno in alunos:", essa é uma abordagem mais limpa e clara do que o programa quer fazer.


(3) como a lista é construída:

Problema identificado em " aprovados = aprovados + [alunos[i]['nome']] " foi a gambiarra para concatenar listas, onde uma lista é criada toda vez para colocar um nome de um aluno e concatenar com uma lista já
existente. É um desperdício de recursos, já que a abordagem padrão e otimizada do python para adicionar elementos em uma lista é o método .append(), exemplo: " aprovados.append(aluno['nome']) ".

Código corrigido:
def processar_alunos_aprovados(alunos): # alteração do nome da função para mostrar o que ela faz de maneira específica, pois estava genérica. Além de retirar a lista vazia como valor padrão para o parâmetro da função, em caso de não ser passada, pois pode gerar erro de vazamento de dados, onde a partir da 2ª chamada da função um argumento pode se misturar com outro
    aprovados = []
    for aluno in alunos:    # o uso do range com índice e len(alunos) foi substituído, dando mais legibilidade e obtendo o mesmo resultado, pois o índice pode ser suprimido de maneira que não seja uma variável desnecessária na memória
        if aluno['nota'] >= 7.0:
            aprovados.append(aluno['nome']) # o uso de append é o padrão para adicionar elementos na lista, a maneira antiga é uma adaptação disso usando concatenação de listas, dificultando a legibilidade.
    print(aprovados)
